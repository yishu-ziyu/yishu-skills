# 开发日志：eve-transcriber Skill 迭代记录

**日期**: 2026-03-28
**主题**: 从视频转录需求到可复用 Skill 框架

---

## 需求

用户希望将 B站视频转成文字稿，便于学习和复盘。

**核心诉求**：
1. 把视频/音频转成 Markdown 文字稿
2. 保留关键信息（元数据、时长等）
3. 方便后续分析和复盘

---

## 解决方案

经过多次迭代，最终确定了**唯一转录方案**：

> **Qwen3-ASR-0.6B 本地模型** - 无需 API key，完全本地运行

### 技术栈

| 组件 | 说明 |
|------|------|
| 模型 | Qwen/Qwen3-ASR-0.6B |
| 运行环境 | eve 项目 venv (Python) |
| 音频处理 | ffmpeg |
| 下载 | yt-dlp |
| 输出 | Markdown |

---

## 遇到的问题及解决

### 问题 1：多种方案尝试导致混乱

**现象**：
- 先尝试 Whisper API → 找不到 API key
- 再尝试本地 Whisper → 装错包（graphite-whisper）
- 浪费时间和精力

**解决**：
- 确定**唯一方案**：只用 Qwen3-ASR 本地模型
- 理念：**少即是多**，不追求多种方案

**教训**：遇到问题先盘点已有资产，不要盲目尝试新方案

---

### 问题 2：长音频转录被截断

**现象**：
- 约 5 分钟的视频转录，只输出一部分
- 原因：Qwen3-ASR 模型有输出长度限制

**解决**：
- 添加**自动分段功能**（>3 分钟自动分 90 秒一段）
- 分段后模型只加载一次，大幅提升速度

**教训**：模型有局限，要通过工程手段绕过

---

### 问题 3：输出路径 bug

**现象**：
- 输出文件跑到 `~//Desktop/即时学习/`（双斜杠）

**解决**：
```bash
# 修复前
OUTPUT_DIR="${2:-$HOME/Desktop/即时学习}"

# 修复后
OUTPUT_DIR=$(eval echo "${2:-$DEFAULT_OUTPUT}")
```

**教训**：bash 变量展开要注意

---

### 问题 4：输出包含噪音标记

**现象**：
- Markdown 输出包含 `[1/4]` `[2/4]` 等进度标记

**解决**：
```bash
# 过滤进度标记
grep -vE '^\[|^Setting|following generation|temperature'
```

**教训**：输出要干净，用户只关心内容

## 从问题到框架

在修复过程中，我意识到：

**问题**：如何评估一个 Skill 是否合格？

**解决方案**：借鉴 AutoResearch 的思想，设计了 Skill Eval Framework

```
┌─────────────────────────────────────────┐
│         SKILL EVAL FRAMEWORK             │
│                                          │
│  1. 定义 evals.json (测试用例)          │
│  2. 运行 Skill                          │
│  3. 检查指标 (pass/fail)                │
│  4. 记录结果                            │
│  5. 迭代优化                            │
└─────────────────────────────────────────┘
```

### 迭代循环

```
EVAL → 发现问题 → FIX → 验证 → 更新 evals.json → PUSH → LOOP
```

### 质量基线

| 指标 | 目标 |
|------|------|
| Pass Rate | > 70% |
| 输出质量 | 干净可用 |

## 总结

这次开发的核心洞见：

1. **少即是多**：不追求多种方案，只保留一个最好的
2. **量化验证**：每个修复都要验证，不能靠感觉
3. **迭代积累**：问题不是一次性解决，而是持续改进
4. **抽象框架**：从具体问题中抽象出可复用的方法论

**最终产出**：
- 一个可用的 Skill（eve-transcriber）
- 一个可复用的框架（Skill Eval Framework）
- 一个可持续的迭代机制（Iteration Process）
