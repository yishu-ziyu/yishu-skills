# /research-report

3-Agent 买方研报撰写工作流：研究检索 → 写作 → 审核 → 图表生成 → 文档组装 → PDF输出。

## 何时使用

用户提及以下场景时自动调用：
- `/research-report`
- "写研报"、"撰写研报"、"开始写报告"
- "帮我写一份XX公司的研究报告"
- "开始XX公司的研究覆盖"
- 任何涉及系统化撰写股票研究报告的场景

## 第二阶段：Agent 1 — 研究检索

### 触发时机
- 每次撰写新部分前必须先完成研究检索
- 撰写过程中发现信息不足时主动触发二次检索

### 检索优先级

```
第一步：本地仓库全面检索（必须先于外部搜索）
├── 公司公告/年报/招股书PDF（精确到页码）
├── 已完成的各部分草稿
├── 历史调研文档
└── 数据溯源清单

第二步：外部搜索（仅本地不足时）
└── 使用 WebSearch / exa-web-search MCP

第三步：矛盾数据专项核查
└── 发现矛盾数据时必须专项追溯原始来源
```

### 检索关键词体系

| 维度 | 关键词组合 |
|------|-----------|
| 财务数据 | "营收"、"收入"、"财务"、"毛利率"、"净亏损" |
| 商业模式 | "Token"、"付费模式"、"订阅"、"交易模式" |
| 产品技术 | "VONE"、"模块"、"平台"、"DOne"、"VOne" |
| 客户结构 | "ARPU"、"客户"、"资管"、"金融" |
| 估值相关 | "估值"、"DCF"、"P/S"、"市值" |
| 竞品对标 | "竞争"、"Palantir"、"恒生"、"Snowflake" |

### Context Bundle 标准输出格式

研究检索完成后，必须产出以下结构：

```markdown
## Context Bundle — 第X部分撰写支撑材料

### 一、可直接引用的数据点（已核实）
- [数据1] 来源：[文件名/URL]，原文摘要，页码（如适用）
- [数据2] 来源：[文件名/URL]，原文摘要，页码（如适用）

### 二、需进一步核实的信息（存疑项）
- [信息1] 现状：现有资料存在矛盾/缺失，建议通过[渠道]核实
- [信息2] 现状：缺乏直接证据，建议[降级处理/删除/标注"待核实"]

### 三、检索结论
- 核心信息完整度：XX%
- 关键数据缺口：[列出2-3项最重要的缺失]
- 建议：本部分撰写是否具备充分信息支撑

### 四、参考资料清单
- [文件/URL1] — 主要引用
- [文件/URL2] — 补充参考
```

### 矛盾数据处理规范

当发现矛盾数据时（如订阅收入占比的"反向变化"），必须：

1. **专项追溯**：找出两个数据的原始来源，明确差异原因
2. **分析原因**：统计口径差异/收入确认差异/时间截点差异
3. **给出明确结论**：哪个数据更可信，或两者都可信但含义不同
4. **标注投资影响**：这个矛盾对投资判断意味着什么

## 第四阶段：Agent 3 — 审核

### 触发时机
收到 Agent 2 的草稿后，必须进行逐条审核。

### 审核标准（逐条检查）

**一、信息准确性**
- [ ] 所有数据是否有明确来源？
- [ ] 来源是否可信（公司公告/权威机构/已核实文件）？
- [ ] 存疑数据是否已降级处理（删除/标注"待核实"）？
- [ ] 数字计算是否准确（百分比、增速、估值）？
- [ ] 货币单位是否全篇统一？

**二、逻辑完整性**
- [ ] 是否回答了本部分的核心问题？
- [ ] 论点与论据是否匹配（结论有数据支撑）？
- [ ] 是否存在逻辑跳跃？
- [ ] 段落之间是否有清晰过渡？
- [ ] 与前序部分是否存在逻辑矛盾？

**三、论证力度**
- [ ] 核心论点是否在段落开头明确给出？
- [ ] 次级论证是否充分展开？
- [ ] 是否有对立的论点被充分讨论？
- [ ] 结论是否适度（不过分夸大，不过于保守）？

**四、写作质量**
- [ ] 标题是否准确反映内容？
- [ ] 是否存在冗余表述？
- [ ] 表格是否必要？
- [ ] 专业术语是否统一？
- [ ] 英文缩写首次出现时是否标注全称？

**五、格式合规**
- [ ] 头部信息是否完整（日期、评级、目标价、上行空间）？
- [ ] 引用标注是否规范（[[序号]]形式）？
- [ ] 文末是否有数据来源清单？
- [ ] 是否有标准免责条款？

### 审核报告标准格式

```markdown
## 审核报告 — 第X部分

### 审核结论
[ 通过 / 需修改后重审 / 不通过 ]

### 严重问题（必须修正）
- [问题1] 位置：[具体段落/位置]，问题：[描述]，建议：[修改方案]

### 一般问题（建议修正）
- [问题1] 位置：[具体段落/位置]，问题：[描述]，建议：[修改方案]

### 亮点（可保留）
- [亮点1] 位置：[具体段落]，理由：[为什么好]

### 综合评价
[100字以内，总结本部分质量]
```

### 质量门槛

每个部分必须同时满足以下条件才能进入下一部分：
1. Agent 3 审核结论：**通过**（无"严重问题"）
2. 核心信息完整度：**≥85%**（Agent 1 评估）
3. 用户确认：**是**

### 5.3 关键修复经验

| 问题 | 原因 | 修复 |
|------|------|------|
| YoY标题数值错误 | 标题写"翻倍"但数据是+19.2% | 标题数值必须与数据一致 |
| Snowflake代码错误 | 用了NOW（ServiceNow） | 股票代码：SNOW |
| fig4 axvline报错 | `transform`参数不支持 | 用`ax.plot()`替代 |
| fig1 UnboundLocalError | 列表推导式变量作用域 | 预计算sub_vals |

### 5.4 Tufte图表精简原则

```
数据是什么？
├── 1-2个大数字 → BigNumberCard（不用图表）
├── <5类对比 → CSS横向进度条（不用Recharts）
├── 时间序列 → 条形图/面积图
└── 多维度数据 → 表格
```
