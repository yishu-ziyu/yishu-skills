# Research to Slides

将研究内容转化为可演示的HTML幻灯片，包含数据可视化、核心记忆点提取、讲解思路。

## 核心理念

**不是做一份"信息堆砌"的PPT，而是讲一个"让人记住几件事"的故事。**

每次汇报真正有意义的听众记忆最多3-5点。PPT是工具，目标是让人走出会议室后记得关键数字、核心观点或行动建议。

## 目录结构

```
research-to-slides/
├── SKILL.md              # 本文件
├── tests/
│   └── evals.json        # 测试用例与断言
├── scripts/
│   └── (预留：辅助脚本)
└── references/
    └── (预留：参考资料)
```

## Action 流水线

每个 Action 独立可执行，有明确的输入、输出与验证条件。

---

### Action 1: 理解主题与目标听众

**输入**：用户模糊需求
**输出**：结构化确认单
**验证**：用户明确确认

#### 收集信息（单次 AskUserQuestion）

```
Question 1 — 主题 (header: "主题"):
研究主题是什么？（如：某公司IPO、某行业分析、某技术趋势）

Question 2 — 听众 (header: "听众", options):
- 领导/上级 — 偏好简洁结论、行动导向
- 投资方 — 偏好数据支撑、风险收益并重
- 同行/专家 — 偏好深度分析、技术细节
- 公众/媒体 — 偏好通俗易懂、有故事性

Question 3 — 目的 (header: "目的", options):
- 决策支持 — 买/不买、做/不做
- 风险认知 — 了解机会与威胁
- 评估参考 — 全面了解某事物
- 共识建立 — 推动团队达成一致

Question 4 — 时长 (header: "时长", options):
- 5分钟 — 极简版，1-2个核心记忆点
- 15分钟 — 标准版，3个核心记忆点
- 30分钟 — 完整版，5个核心记忆点

Question 5 — 内容 (header: "内容", options):
- 有资料/文档 — 请分享，我会提取关键信息
- 有大纲/笔记 — 请分享，我会补充细节
- 仅有主题 — 我来帮你梳理框架
```

#### 输出格式

```markdown
## 确认单

| 字段 | 值 |
|------|-----|
| 主题 | ... |
| 听众 | ... |
| 目的 | ... |
| 时长 | ... |
| 内容 | ... |
| 记忆点数量 | N个 |
```

**验证条件**：用户回复"确认"或修改后确认。未确认不进入 Action 2。

---

### Action 2: 并行搜索最新资料

**输入**：确认单 + 主题关键词
**输出**：三份独立研究报告
**验证**：数据完整性与时效性（最近6-12个月）

#### Agent A：公司与产品研究
- **搜索范围**：目标公司/主题的官方发布、最新产品、融资历史、估值数据
- **关键词**：公司名 + 产品名 + 融资 + 估值 + 最新动态
- **时间范围**：最近6-12个月
- **输出格式**：
  ```markdown
  ## 公司概览
  - 公司名：
  - 估值：
  - 融资轮次：
  - 最新产品：
  - 关键数据点：
  ```

#### Agent B：市场与竞争研究
- **搜索范围**：行业趋势、竞争对手对比、市场份额、监管动态
- **关键词**：行业名 + 竞争格局 + 市场份额 + 监管 + 政策
- **时间范围**：最近6个月
- **输出格式**：
  ```markdown
  ## 行业概览
  - 市场规模：
  - 主要竞争者：
  - 市场份额分布：
  - 监管动态：
  ```

#### Agent C：风险与机会研究
- **搜索范围**：风险因素、挑战、负面新闻、潜在问题
- **关键词**：公司名 + 风险 + 挑战 + 问题 + 负面 + 诉讼 + 监管
- **时间范围**：最近12个月
- **输出格式**：
  ```markdown
  ## 风险与机会
  - 主要风险：
  - 潜在挑战：
  - 市场机会：
  ```

**执行策略**：3个Agent并行启动，独立产出，汇总到主Agent。

**验证条件**：
- [ ] 每份报告包含最近6-12个月数据
- [ ] 关键数字有来源标注
- [ ] 无严重矛盾信息（若有，需标注争议点）

---

### Action 3: 提取核心记忆点

**输入**：三份研究报告
**输出**：N个记忆点列表（含格式）
**验证**：每个记忆点满足至少2项标准

#### 记忆点格式

| 类型 | 格式 | 例子 |
|------|------|------|
| 数字+对比 | $X — 超沙特阿美 | "$2.27万亿 — 超过沙特阿美" |
| 悖论感 | 月营收X / 年亏损Y | "月营收$20亿 / 年亏损$140亿" |
| 监管信号 | 人物+动作 — 事件 | "Bessent+Powell联名开会 — 史无前例" |
| 速度感 | 从X到Y：用时Z | "从发布到1亿用户：ChatGPT用时2个月，GPT-5仅用3周" |

#### 记忆点质量标准

每个记忆点必须满足至少2项：
1. 有具体数字支撑
2. 有对比参照物
3. 能引发好奇或震惊

#### 用户确认 checkpoint

展示记忆点列表，让用户操作：
- 确认/修改优先级排序
- 删除不感兴趣的点
- 添加用户自己知道的点
- 调整总数量（增加/减少）

**验证条件**：
- [ ] 每个记忆点满足质量标准
- [ ] 用户已确认最终列表
- [ ] 列表不超过 Phase 1 设定的数量上限

---

### Action 4: 生成HTML幻灯片

**输入**：记忆点列表（用户已确认）+ 确认单
**输出**：`output.html` + `speaking-notes.md`
**验证**：比例正确 / 数据来源标注 / 风格一致

#### 设计规范

**字体**：
- 标题：Cormorant Garamond
- 正文：Source Serif 4

**配色**：
- 背景：#faf9f7 (cream)
- 主文字：#1a1a1a
- 强调色：#c41e3a (crimson)

**动画**：
- 数字滚动（count-up animation）
- 元素渐入（fade-in with stagger）
- 进度条动态填充

#### 幻灯片结构

```
Slide 1: 封面
  - 大标题
  - 副标题（核心公司/主题）
  - N个记忆点预览（简短版）

Slide 2: 核心数据总览
  - 关键数字卡片（4-6个）
  - N个记忆点高亮卡片（emphasis cards）
  - 估值对比柱状图

Slide 3 到 Slide N+2: 各公司/主题详情
  - 公司名称 + 估值
  - 核心产品（2-3个最热产品）
  - 关键数据
  - 支持记忆点的数据

Slide N+3: 对比/风险
  - 多维度对比表格（可交互排序）
  - 风险热力图矩阵
  - 环形图（展示结构占比）

Slide N+4: 宏观影响
  - 行业影响
  - 监管动态

Slide N+5: 结论
  - N个记忆点回顾
  - 数据来源
```

#### 数据可视化组件

| 组件 | 用途 | 实现方式 |
|------|------|---------|
| 柱状图 | 估值/融资对比 | 纯CSS，宽度比例精确对应数值 |
| 环形图 | 收入结构占比 | CSS conic-gradient |
| 时间线 | 融资历程/里程碑 | CSS实现 |
| 进度条 | 增长轨迹 | CSS progress bar |
| 热力图 | 风险矩阵 | CSS grid + 颜色 |
| 交互表格 | 多维度对比 | HTML + JS排序 |
| 数字卡片 | 关键指标 | CSS + JS count-up |

**验证条件**：
- [ ] 数字比例正确（300亿 ≠ 600亿，视觉长度不同）
- [ ] 产品信息来自最新搜索结果
- [ ] 每个数据有来源标注
- [ ] 风格保持 Paper & Ink 一致

---

### Action 5: 交付与审计

**输入**：生成物 + 用户反馈
**输出**：最终文件 + 使用记录
**验证**：记录用于版本迭代

#### 交付内容

1. **HTML演示文稿** — `output.html`
   - 可在浏览器直接打开
   - 支持键盘导航（方向键）
   - 支持inline编辑（按E进入编辑模式）

2. **讲解思路** — `speaking-notes.md`
   - 每页要点
   - 时间分配
   - 转折话术

#### 审计记录

每次交付后，记录以下信息用于版本迭代：

```markdown
## 审计记录

| 字段 | 值 |
|------|-----|
| 日期 | ... |
| 主题 | ... |
| 记忆点数量 | N |
| 用户反馈 | ... |
| 问题点 | ... |
| 版本 | v1.0 |
```

**验证条件**：
- [ ] 用户已接收文件
- [ ] 审计记录已更新
- [ ] 用户反馈已记录（即使为空）

## 迭代链路

```
初始推理（Action 1-4）
    ↓
用户中途介入 ← 用户可在任一 checkpoint 干预
    ↓
错误检测（验证条件未满足）
    ↓
反馈注入（修正记忆点/数据/格式）
    ↓
再推理（重新生成受影响的 Slide）
    ↓
收敛（通过验证条件）→ Action 5 交付
```

---

## 用户介入点

| 介入点 | 时机 | 用户可以做什么 |
|--------|------|--------------|
| Action 1 确认后 | 搜索前 | 修改主题、调整听众/目的/时长 |
| Action 2 执行中 | Agent运行时 | 提供额外关键词、排除某些来源 |
| Action 3 记忆点提取后 | 生成PPT前 | 确认/修改/增删记忆点 |
| Action 4 PPT生成后 | 交付前 | 修改具体页面、调整样式 |

## 质量检查清单（交付前必查）

- [ ] N个记忆点清晰、有冲击力

- [ ] 产品信息为最新版本
- [ ] 数据有来源标注
- [ ] 风格保持 Paper & Ink 一致
- [ ] 每页内容不超出密度限制
- [ ] 审计记录已更新

## 常见错误

1. **堆砌信息** — 忘了PPT的目的是让人记住N件事，不是展示所有信息
2. **比例错误** — 100亿:300亿:600亿 = 1:3:6，视觉长度必须对应
3. **产品过时** — 发布前确认产品信息来自最新搜索结果
4. **缺少来源** — 听众可能质疑数据，必须标注来源
5. **跳过 checkpoint** — 用户未确认就直接进入下一步

---

## 版本迭代规则

- 每次用户反馈记录在审计文件中
- 当同一问题被反馈≥3次，启动 skill 重构
- 重构后版本号 +0.1（如 v1.0 → v1.1）
- 低效版本保留 2 个迭代周期后归档
