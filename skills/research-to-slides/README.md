# Research to Slides

将研究内容快速转化为可演示的 HTML 幻灯片。

## 为什么做这个项目

日常工作中，我们经常需要把研究内容分享给他人。但传统 PPT 存在三个问题：

1. **信息过载** — 把所有数据都堆上去，听众走出会议室什么都记不住
2. **制作耗时** — 排版、动画、数据可视化，每一项都要耗费大量时间
3. **缺乏记忆点** — 没有核心观点的串联，汇报变成流水账

我们希望解决这三个问题：**让人走出会议室后，记得几件关键的事。**

## 方法实现

### 核心方法论：Action 流水线 + 迭代验证

```
用户需求 → Action 1 确认 → Action 2 并行搜索
    → Action 3 提取记忆点 → Action 4 生成幻灯片
    → Action 5 交付 + 审计记录
```

每一步都有验证条件，未通过不进入下一步。用户可在任意 checkpoint 中途介入，修改方向或补充信息。

### 关键设计原则

| 原则 | 说明 |
|------|------|
| **记忆点驱动** | 不是堆砌信息，而是围绕 N 个核心记忆点组织内容 |
| **数字有比例** | 300 亿和 600 亿在图表中长度必须不同 |
| **数据有来源** | 每个数字都能追溯，听众质疑时有据可查 |
| **用户全程可控** | 每个 phase 结束后有确认点，不满意随时修改 |

### 技术实现

- **纯 HTML** — 无需安装任何软件，浏览器直接打开
- **零依赖** — 无 npm、无构建工具、所有 CSS/JS 内联
- **Paper & Ink 美学** — Cormorant Garamond + Source Serif 4 字体，cream + crimson 配色
- **数据可视化** — 纯 CSS 实现柱状图、环形图、时间线、热力图、交互表格

## 最终结果

交付两个文件：

1. **output.html** — 可演示的 HTML 幻灯片
   - 键盘方向键导航
   - 按 E 进入 inline 编辑模式
   - 数字滚动动画、数据可视化图表

2. **speaking-notes.md** — 讲解思路文档
   - 每页要点
   - 时间分配建议
   - 转折话术

## 方法论来源

本技能的设计参考了以下原则：

- **AutoResearch 验证与迭代流程** — 明确的 Action 列表、输入输出格式、验证条件
- **用户介入机制** — 允许在每个 phase 之间中途干预，而非线性执行到底
- **审计与迭代** — 每次使用后记录反馈，用于后续版本优化
